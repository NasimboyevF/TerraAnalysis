# 🌱 Система мониторинга деградации земли

Клиент-серверное приложение для сбора, хранения и анализа данных о состоянии земельных участков.

## Стек
- **Frontend:** React (Vite) + Tailwind CSS + Zustand + Recharts + Leaflet
- **Backend:** Node.js + Express.js + MongoDB + Mongoose + JWT
- **Telegram Bot:** Telegraf.js (работает в одном процессе с backend)

📄 API: смотри [`postman_collection.json`](./postman_collection.json) — импортируется прямо в Postman
🚀 Деплой: пошаговая инструкция в [`DEPLOY.md`](./DEPLOY.md)

## 🚀 Запуск проекта

### 1. Убедитесь что MongoDB запущена

```bash
# Linux / Mac
sudo systemctl start mongod

# Windows — откройте MongoDB Compass или запустите службу mongod
```

MongoDB автоматически создаст базу данных при первом запуске.
Данные хранятся локально на вашем компьютере (обычно в /var/lib/mongodb).

### 2. Настройте Backend

```bash
cd backend
cp .env.example .env
# Откройте .env и при необходимости поменяйте JWT_SECRET
npm install
npm run dev
```

Сервер запустится на http://localhost:5000

### 3. Настройте Frontend

```bash
cd frontend
npm install
npm run dev
```

Приложение откроется на http://localhost:5173

## 🤖 Telegram Bot

Бот запускается автоматически вместе с backend (`npm run dev` в папке `backend`),
если в `.env` задан `BOT_TOKEN`.

### Настройка

1. Создать бота через [@BotFather](https://t.me/BotFather), получить токен
2. Прописать `BOT_TOKEN=...` в `backend/.env`
3. Запустить backend — в консоли появится `🤖 Telegram-бот запущен в режиме polling`
4. Написать боту `/start`, затем привязать аккаунт: `/link email пароль`

В проде (`BOT_WEBHOOK_URL` задан в `.env`) бот работает через webhook —
Express сам поднимает роут `/bot-webhook`.

### Команды пользователя

| Команда | Описание |
|---|---|
| `/start` | Приветствие |
| `/link email пароль` | Привязать Telegram к аккаунту сайта |
| `/help` | Список команд |
| `/info` | Профиль привязанного пользователя |
| `/items` (или `/products`) | Список своих образцов земли с пагинацией |
| `/addsample` | Пошаговый мастер (wizard) добавления образца прямо в чате |

### Команды администратора

Доступны только пользователям с `role: 'admin'` (см. ниже, как назначить первого админа).

| Команда | Описание |
|---|---|
| `/admin` | Открыть меню админ-панели (inline-кнопки) |
| `/stats` | Статистика: пользователи, образцы, отчёты, разбивка по уровню деградации |
| `/users` | Список пользователей с ролями и статусом привязки Telegram |
| `/broadcast текст` | Разослать сообщение всем привязанным пользователям |
| `/promote email` | Назначить другого пользователя администратором |

### Как назначить первого админа

Ролей по умолчанию нет ни у кого (все регистрируются как `user`). Первого админа
проще всего назначить напрямую в MongoDB после регистрации:

```js
// mongosh
use land-degradation
db.users.updateOne({ email: "тебя@example.com" }, { $set: { role: "admin" } })
```

Дальше можно раздавать роль другим через `/promote email` в самом боте.

### Уведомления

- При регистрации нового пользователя — уведомление всем привязанным админам
- При создании отчёта — уведомление владельцу образца, а при критическом уровне
  деградации — дополнительно всем админам

## 🖥️ Frontend — новые страницы

- **/profile** — просмотр и редактирование имени/email, статус привязки Telegram, роль
- **/admin** (только для `role: 'admin'`) — статистика системы, разбивка отчётов по уровню
  деградации, таблица пользователей с назначением/снятием роли администратора
- Все страницы подключены через `React.lazy` + `Suspense` — каждая грузится отдельным
  чанком только когда пользователь на неё переходит

## 🧮 Формула деградации

```
degradationScore = NDVI(35%) + SQI(30%) + moisture(20%) + organicMatter(15%)
```

- **0–25** → низкий уровень (🟢)
- **26–50** → умеренный (🟡)
- **51–75** → высокий (🟠)
- **76–100** → критический (🔴)
